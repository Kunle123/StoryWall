# StoryWall — product roadmap

**Purpose:** Direction and tradeoffs. **Execution** lives in **GitHub Issues + Projects**; narrative detail in [`LAUNCH_VALIDATION_STRATEGY.md`](./LAUNCH_VALIDATION_STRATEGY.md).

## Launch posture (2026)

- **Now:** **Private beta** / targeted validation — not a broad public launch.
- **Goal this month:** Prove understanding, quality output, share, repeat use — for **one wedge** (see strategy doc).
- **Broad launch** only after gates in [#14](https://github.com/Kunle123/StoryWall/issues/14) (or conscious deferral).

## Launch goals

- Validate **current affairs / history explainers** as primary wedge (secondary: teachers/students).
- Ship **homepage + messaging** that matches the wedge [#15](https://github.com/Kunle123/StoryWall/issues/15), aligned with [#7](https://github.com/Kunle123/StoryWall/issues/7) where UI work is needed.
- Run **Week 1–4** execution tracks in [`current-sprint.md`](./current-sprint.md) and linked issues **#14–#24**.

## This quarter — priorities (ordered)

1. **Validation & learning** — analytics [#17](https://github.com/Kunle123/StoryWall/issues/17), feedback [#18](https://github.com/Kunle123/StoryWall/issues/18), user tests [#19](https://github.com/Kunle123/StoryWall/issues/19), founder distribution [#20](https://github.com/Kunle123/StoryWall/issues/20).
2. **Product clarity** — examples + How it works + demo [#16](https://github.com/Kunle123/StoryWall/issues/16); free-tier messaging [#21](https://github.com/Kunle123/StoryWall/issues/21).
3. **Engineering backlog** (when validation allows) — see **#4–#13** (tokens, collections, leaderboard, infra, etc.).
4. **Technical SEO & share hygiene** — [#25](https://github.com/Kunle123/StoryWall/issues/25) (GSC, sitemap monitoring, OG re-scrape); complements **#15–#17** but is **not** a large SEO campaign (see strategy doc).

## What we are not building now

- Broad **paid ads**, big **PR** spends, **SEO programs** as primary lever — see [#22](https://github.com/Kunle123/StoryWall/issues/22) (budget guardrails).
- **Monetization optimization** as the main focus this month — [#21](https://github.com/Kunle123/StoryWall/issues/21).
- **General-purpose “everyone who likes storytelling”** positioning — [#15](https://github.com/Kunle123/StoryWall/issues/15).

## Target users (current bet)

- **Primary wedge:** Current affairs / history **explainers** (news, clarity, shareability).
- **Secondary:** Teachers, students, revision timelines.
- Not optimizing for “all creatives” in the near term.
